  3. المصادقة والأدوار
═══════════════════════════════════════════════════════════

نظام المصادقة:
- مصادقة مخصصة (Custom) — ليست NextAuth
- كلمة المرور مشفّرة بـ bcryptjs (10 rounds)
- الجلسة تُخزَّن في جدول Session بقاعدة البيانات (تدوم 30 يوماً)
- Cookie: rcs-session (httpOnly, sameSite=lax, secure in production)
- عند أول تشغيل: ينشئ تلقائياً حساب admin@rcs.dz / admin123

الأدوار الأربعة:
  الدور                 | الرمز  | الصلاحيات
  مدير (admin)          | 👑    | كل الصلاحيات
  مساعد إداري (assistant)| 💼   | منخرطون، حضور، تجديد، تأمين، بطاقات، استيراد، تصدير، ساعات عمل
  حارس سباحة (lifeguard)| 🏊    | لوحة التحكم، الحضور، ساعات عمل (سجلاته فقط)
  مراقب (observer)      | 👁️   | لوحة التحكم، الحضور (عرض فقط)

حسابات افتراضية:
- admin@rcs.dz / admin123 (مدير)
- assistant@rcs.dz / assistant123 (مساعد)
- coach@rcs.dz / coach123 (حارس سباحة)
- observer@rcs.dz / observer123 (مراقب)

دخول الكاشير السريع (PIN):
- صفحة /pin منفصلة
- رمز 4 أرقام بدون اسم مستخدم/كلمة مرور
- لوحة مفاتيح كبيرة على الشاشة
- يدعم لوحة المفاتيح الفيزيائية
- أصوات تأكيد (نجاح/فشل)
- يمكن إنشاء عدة أكواد PIN لكل كاشير/تابلت
- كل PIN يحدد دوره (مساعد/حارس/مدير)
- جدول CashierPin في قاعدة البيانات

الموافقة على الحسابات:
- المستخدمون الجدد يمكن أن يُسجّلوا بحالة "pending"
- المدير يرى الحسابات المعلّقة ويوافق عليها مع تحديد الدور

نظام الفئات العمرية (4 فئات بحد 13 سنة):

  ذكور أقل من 13 سنة       | males_under_13    | #0ea5e9 (سماوي)

حساب الرسوم:
  الحالة                              | الرسوم
  نوع MJ                              | 0 دج (لا يدفع شيئاً، لا يأخذ رقم عضوية)
  FCS / RCS / تأمين فقط               | 0 دج (معفى)
  OPOW / DJS / POLICE / اشتراك 300   | 300 دج
  عادي، أقل من 14 سنة                 | 1300 دج
  عادي، 14 سنة فأكثر                  | 1500 دج
  مصاريف التأمين                      | 500 دج (MJ معفى)
  حقوق المركب                         | 1000 دج (MJ والأنواع الخاصة معفاة)

المبلغ الإجمالي = رسوم الاشتراك + مصاريف التأمين
(حقوق المركب مستثناة من الإجمالي)

حالة التجديد:
- ✅ ساري (أكثر من 5 أيام متبقية)
- ⚠️ قريب الانتهاء (5 أيام أو أقل)
- ⛔ منتهي - يتطلب تجديد
- 🔒 مجمدة (لم يدفع)

تاريخ انتهاء الاشتراك:
- آخر دفعة + 30 يوم

تسعير ساعات العمل:
- يحددها المدير من الإعدادات (افتراضي: 200 دج/ساعة)
- المستحق = إجمالي الساعات المعتمدة × سعر الساعة
- المتبقي = المستحق - المسدد

═══════════════════════════════════════════════════════════
  8. النظام الهجين (محلي + سحابي)
═══════════════════════════════════════════════════════════

المعمارية:
  كمبيوتر (Electron)  ←→  سحابة (Vercel)  ←→  هاتف (PWA)
       SQLite/Neon         Neon PostgreSQL       IndexedDB
       أوفلاين كامل         المصدر الموثوق         أوفلاين كامل

Service Worker v4:
- HTML: network-only (لا كاش) — يمنع "Failed to load chunk"
- Chunks (_next/static): pass-through (browser HTTP cache)
- API GET: network-first → cache → empty-JSON fallback
- صور: stale-while-revalidate
- QR images: cache-first
- Auto-reload عند فشل chunk

Fetch Interceptor (inline في <head>):
- يُحقن قبل React — يلفّ window.fetch
- أوفلاين + GET → JSON فارغ (200) لكل endpoint
- أوفلاين + POST/PUT/DELETE → IndexedDB outbox + JSON نجاح (200)
- مستحيل رؤية خطأ أحمر عند العمل أوفلاين

محرك المزامنة (Sync Engine):
- Pull: يجلب آخر التغييرات من السحابة
- Push: يرسل outbox entries
- حل التعارضات: last-write-wins
- تلقائي: كل 5 دقائق + عند عودة الإنترنت
- يدوي: زر المزامنة

مؤشر المزامنة (SyncIndicator):
- 🟢 متصل + متزامن
- 🟡 جاري المزامنة...
- 🔴 غير متصل (أوفلاين)
- وقت آخر مزامنة
- لوحة قابلة للتوسيع مع تفاصيل

تطبيق Electron:
- يحمل التطبيق السحابي مباشرة (CLOUD_URL)
- 155MB مضغوط — بدون Node.js أو native modules
- قوائم: ملف (طباعة/تحديث/خروج)، تحرير، عرض، مساعدة
- اختصارات: Ctrl+P, Ctrl+R, Ctrl+Q, F11, F12

  10. الأصوات (Web Audio API)
═══════════════════════════════════════════════════════════

- لا تحتاج ملفات صوت خارجية — تُولّد بـ OscillatorNode
- نجاح ✓: نغمتان صاعدتان (880Hz → 1320Hz)
- تحذير ✗: نغمتان هابطتان (220Hz → 180Hz, square wave)
- نقرة: 660Hz قصيرة (triangle)
- خطأ: 300Hz → 220Hz (sawtooth)
- إشعار: ثلاث نغمات صاعدة (988 → 1318 → 1568)
- تُشغّل عند: تسجيل حضور (نجاح/تحذير)، حذف، خطأ

═══════════════════════════════════════════════════════════
  19. مبادئ التصميم المتبعة
═══════════════════════════════════════════════════════════

1. Mobile-First — الكود يُكتب للهاتف أولاً، ثم sm:/lg: للشاشات الأكبر
2. auto-fit grids — المتصفح يحدد عدد الأعمدة تلقائياً
3. transform: scale — للعناصر ذات الأبعاد الثابتة (ترويسة، بطاقات)
4. Offline-first — كل عمليات الكتابة تُخزّن محلياً وتُزامن لاحقاً
5. Zero red errors — fetch interceptor يرجع نجاح وهمي عند انقطاع الإنترنت
6. React.memo — لمنع إعادة الرسم غير الضرورية
7. CSS contain: content — لعزل البطاقات عن بعضها
8. Service Worker v4 — لا يكاش HTML (يمنع chunk errors)
9. Self-healing — ensureDefaultAdmin + ensureDefaultSettings عند كل تسجيل دخول
10. last-write-wins — حل التعارضات في المزامنة
